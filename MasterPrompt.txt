# Agent Development Guidelines & Style Guide
This document defines the coding standards, architectural patterns, and style guidelines that must be strictly followed for all development tasks in this project.

## 1. SLO Agent Skills Specification
All agent-based functionality must be built according to the **Service Level Objective (SLO) Agent Skills** framework.

* **Skill Definition:** Ensure every skill has a clearly defined input schema, output schema, and error handling mechanism.

TypeScript Frontend/Node

## 3. General Principles
* **Documentation:** Every public method and class must be documented.
* **Testing:** Write unit tests for all new logic. Use type-driven development to catch errors at compile/lint time.
